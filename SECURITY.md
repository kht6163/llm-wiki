# 보안 정책

## 지원 범위

보안 수정은 최신 정식 릴리즈를 기준으로 제공합니다. 이전 버전에서 문제가 재현되면 먼저
최신 버전으로 업그레이드한 뒤 다시 확인해 주세요.

## 취약점 신고

공개 이슈에 취약점 세부 정보, API 키, 세션 쿠키, 문서 원문을 올리지 마세요. GitHub의
[비공개 보안 권고](https://github.com/kht6163/llm-wiki/security/advisories/new)로 다음 정보를
보내 주세요.

- 영향을 받는 버전과 배포 방식
- 재현 절차와 예상 영향
- 가능한 경우 최소 재현 코드 또는 요청/응답(비밀 값은 제거)
- 이미 알려진 완화 방법

접수 후에는 재현 여부와 수정 범위를 확인하고, 수정 릴리즈가 준비될 때까지 세부 내용을
비공개로 유지합니다. 자격 증명이 노출된 경우 신고와 별개로 즉시 비밀번호를 변경하고
관련 API 키를 폐기하세요.

## 운영자 기본 수칙

- 웹과 MCP 포트는 기본처럼 루프백 또는 신뢰 네트워크에만 노출하고 외부 공개는 TLS
  리버스 프록시를 통합니다.
- `/metrics`는 인증이 없으므로 스크레이프 네트워크 밖에서 차단합니다.
- 릴리즈 전 snapshot을 만들고 복원 훈련을 정기적으로 수행합니다.
- 의존성 및 컨테이너 취약점 경고를 확인하고 지원 중인 최신 릴리즈를 사용합니다.
